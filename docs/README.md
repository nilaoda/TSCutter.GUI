## 共同前提

当前三个工具均以标准 188 字节 MPEG-TS 包为处理对象。

共同的基础概念包括：

- **TS 包**：固定 188 字节，包含同步字节、PID、连续计数器、适配字段和负载。
- **PID**：用于区分节目表、视频、音频、字幕、时钟及其他数据。
- **PSI/SI**：PAT、PMT、SDT、EIT 等节目与业务信息。
- **PES**：承载音视频等基本流，并可包含 PTS、DTS 等时间戳。
- **ES**：去除 TS 和 PES 封装后的音视频基本流字节。
- **PCR**：节目时钟参考，用于建立播放时间线和复用时序。

## 设计原则

三个工具共享以下原则：

1. **流式处理**：不将完整媒体文件读入内存。
2. **性能优先**：以顺序读取、低分配和有限状态分析为主。
3. **保守判断**：信息不足时宁可少报、少修，也不伪造无法确认的数据。
4. **不做深度解码**：主要检查封装、连续性、时间戳和码流边界，不进行完整音视频解码。
5. **输出可解释**：异常、自动依赖和修复结果都应能说明原因。
6. **内存有界**：事件、时间轴、指纹与候选数据均应设置上限或压缩策略。

## 文档边界

这些文档描述当前方案的设计意图，不等同于 MPEG-TS 标准全文。涉及广播规范、编码语法或地区性扩展时，应以对应标准为最终依据。
